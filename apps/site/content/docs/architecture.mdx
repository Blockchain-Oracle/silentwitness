## Eight boundaries — six in code, two supplementary

**In code (architectural):**
1. **Read-only evidence mount** — `/evidence ro,noexec,nosuid` blocks any write
   to source artifacts.
2. **MCP toolset firewall** — per-specialist toolset; memory specialist cannot
   reach disk tools, disk specialist cannot reach network tools.
3. **Entity gate** — agent-claimed entities must resolve in `index.db` before
   an observation is recorded.
4. **Citation gate** — every finding carries ≥1 `cited_span` resolving to an
   audit JSONL row.
5. **Hash-chained audit ledger** — `record_hash` + `prev_record_hash` per row;
   `silentwitness verify --audit-chain` walks every backend.
6. **HMAC approval ledger** — `silentwitness approve` adds an HMAC-SHA256 entry
   that downstream systems can verify out-of-band.

**Supplementary (prompt-based):**
1. Investigator system prompt — encodes domain priors.
2. Critic agreement prompt — encodes self-doubt heuristics.

Removing the supplementary prompts degrades quality but **does not unlock
hallucinations against unmounted artifacts**. That's the whole point.

## Per-component reference

- **MCP server** — [`src/silentwitness_mcp/`](https://github.com/Blockchain-Oracle/silentwitness/tree/main/src/silentwitness_mcp). 12 agent-visible tools over parsed evidence records, findings, and audit output.
- **Agent + critic** — [`src/silentwitness_agent/`](https://github.com/Blockchain-Oracle/silentwitness/tree/main/src/silentwitness_agent). Pydantic AI 1.105+, `request_limit=None`.
- **Audit chain** — [`src/silentwitness_mcp/audit/chain.py`](https://github.com/Blockchain-Oracle/silentwitness/blob/main/src/silentwitness_mcp/audit/chain.py). Per-path lock (threading + fcntl) + cross-process cache invalidation.
- **Verification gates** — [`src/silentwitness_mcp/verification/`](https://github.com/Blockchain-Oracle/silentwitness/tree/main/src/silentwitness_mcp/verification/). Entity gate, citation gate, sanitizer.
