## File modification map

- `.pre-commit-config.yaml` — NEW — verbatim from CICD_SPEC §3.2: ruff-format + ruff + mypy + file-size-guard + forbidden-paths + detect-secrets + standard hooks + conventional-pre-commit (~85 LOC).
- `.pre-commit-hooks/file-size-guard.py` — NEW — verbatim from CICD_SPEC §6.1 (~80 LOC). Counts ALL physical lines (blanks + comments). Skips `uv.lock`, `*.json`, `*.lock`, `tests/**/fixtures/*`, `vendored/*`, `.pre-commit-hooks/*`. Exits 1 on any file >400 LOC.
- `.pre-commit-hooks/forbidden-paths.py` — NEW — verbatim from CICD_SPEC §6.2 (~75 LOC). Blocks writes to `evidence/*`, `var/lib/silentwitness/*`, `cases/*` (with `tests/integration/fixtures/` carve-out).
- `.secrets.baseline` — NEW — generated by `uv run detect-secrets scan > .secrets.baseline` from an empty tree.
- `tests/unit/test_pre_commit_hooks.py` — NEW — 8 behavioral tests: file-size-guard returns 0 on 400-LOC file, returns 1 on 401-LOC file, skips `uv.lock`, skips fixture path; forbidden-paths returns 0 on `src/foo.py`, returns 1 on `cases/case-001/audit/x.jsonl`, returns 1 on `evidence/raw.E01`, returns 0 on `tests/integration/fixtures/cases/sample/audit/x.jsonl` (carve-out).

## Acceptance criteria (BDD — machine-verifiable)

```
Given .pre-commit-config.yaml exists with the hook list from CICD_SPEC §3.2
When  `uv run pre-commit install --install-hooks --hook-type commit-msg --hook-type pre-commit` runs
Then  exit code is 0
And   `.git/hooks/pre-commit` and `.git/hooks/commit-msg` exist

Given .pre-commit-hooks/file-size-guard.py exists
When  a 400-line .py file is passed
Then  exit code is 0

Given .pre-commit-hooks/file-size-guard.py exists
When  a 401-line .py file is passed
Then  exit code is 1
And   stderr contains "401" and the file path
And   stderr contains "split at a natural module boundary"

Given .pre-commit-hooks/forbidden-paths.py exists
When  the path "cases/case-001/audit/foo.jsonl" is passed
Then  exit code is 1
And   stderr contains "cases/case-001/audit/foo.jsonl"

Given .pre-commit-hooks/forbidden-paths.py exists
When  the path "tests/integration/fixtures/cases/sample/audit/foo.jsonl" is passed
Then  exit code is 0 (carve-out honored)

Given .pre-commit-hooks/forbidden-paths.py exists
When  the path "src/silentwitness_mcp/server.py" is passed
Then  exit code is 0

Given the hook chain is installed
When  `uv run pre-commit run --all-files` runs on the freshly scaffolded tree
Then  exit code is 0 (no offenders before product code lands)

Given tests/unit/test_pre_commit_hooks.py exists
When  `uv run pytest tests/unit/test_pre_commit_hooks.py -v` runs
Then  exit code is 0
And   8 tests pass

Given the conventional-pre-commit hook is wired
When  `git commit -m "feat: foo"` runs (allowed types only, no scope required)
Then  exit code is 0

Given the conventional-pre-commit hook is wired
When  `git commit -m "garbage message"` is attempted (with hook bypass disabled)
Then  exit code is 1
```

## Notes for coding agent

- Reference: CICD_SPEC.md §3 (full `.pre-commit-config.yaml`), §6.1 (file-size-guard.py verbatim), §6.2 (forbidden-paths.py verbatim).
- Copy both hook scripts character-for-character from CICD_SPEC §6.1 and §6.2. They are reference implementations; do NOT improvise.
- The file-size-guard threshold is **400 LOC**, ALL physical lines (blanks + comments included). This is intentional — encourages aggressive splitting rather than whitespace gaming. Documented in CICD_SPEC §6.1 docstring.
- Skip patterns in `file-size-guard.py` include `.pre-commit-hooks/*` itself (the hook is tooling, exempted from the rule it enforces). Do not remove this — `forbidden-paths.py` is also under `.pre-commit-hooks/` and is ~75 LOC, far under the limit, but the exemption is structural.
- `forbidden-paths.py` carve-out is exactly `tests/integration/fixtures/` — no other prefix. Fixtures shipped for integration testing live here; everything else under `cases/`, `evidence/`, or `var/lib/silentwitness/` is a hard block.
- The conventional-pre-commit allowed scopes are: `mcp`, `agent`, `hypothesis`, `report`, `verify`, `audit`, `harness`, `docs`, `ci`, `deps` (CICD_SPEC §7.2). Scope is **optional** in the pre-commit hook config (commit messages may include or omit scope).
- `.secrets.baseline` should be generated last, AFTER all other files are in the tree, by running `uv run detect-secrets scan > .secrets.baseline` and committing the result. The baseline file is excluded from the detect-secrets hook itself (CICD_SPEC §3.2 `exclude` block).
- DO NOT run pre-commit hooks with `--no-verify` to bypass failures. If a hook fails on this story's own commits, fix the underlying issue.
- Library docs to consult via Context7 BEFORE writing:
  - `pre-commit` topic `local hooks language: system` (the entry point convention for `uv run python <script>`).
  - `detect-secrets` topic `baseline` (the `--baseline` invocation pattern in v1.5.0).
